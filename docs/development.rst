Development guide
=================

Build the documentation
-----------------------

Install the project with the optional documentation dependencies:

.. code-block:: console

   pip install -e ".[dev]"

Build a local HTML site from the repository root:

.. code-block:: console

   sphinx-build -b html docs docs/_build/html

Open ``docs/_build/html/index.html`` after a successful build. To make
warnings fail the build in continuous integration, use ``-W``:

.. code-block:: console

   sphinx-build -W -b html docs docs/_build/html

Documenting code
----------------

The API pages are generated by Sphinx's ``autodoc`` extension. It reads
Python docstrings and type annotations directly from ``mckit``. Keep the
signature typed and add a docstring whenever a function, method, or class is
added or its behavior changes.

Use Google-style sections for public callables when their parameters or return
value need explanation:

.. code-block:: python

   def build_structure(symbol: str, repetitions: int = 1) -> Atoms:
       """Build a periodic structure.

       Args:
           symbol: Chemical symbol for the structure.
           repetitions: Number of repeated unit cells.

       Returns:
           The generated atomic structure.

       Raises:
           ValueError: If ``repetitions`` is less than one.
       """

The configuration deliberately includes private members. This makes internal
helpers discoverable to contributors without making them part of the package's
supported public API.
